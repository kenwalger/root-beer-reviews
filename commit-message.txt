Add comprehensive Sphinx docstrings and type hints to all Python files

This PR adds Sphinx-style docstrings and type hints throughout the entire
codebase to improve code documentation, IDE support, and maintainability.

Key Changes:
- **Module Documentation**: Added module-level docstrings to all Python files
  - Explains purpose, usage, and key concepts for each module
  - Documents module structure and dependencies

- **Function Documentation**: Added Sphinx-style docstrings to all functions
  - Parameter descriptions with types (:param, :type)
  - Return value documentation (:returns, :rtype)
  - Exception documentation (:raises)
  - Usage examples and notes where appropriate

- **Type Hints**: Added comprehensive type hints throughout
  - Function parameters and return types
  - Class attributes and methods
  - Type hints for async functions and generators
  - Union types for optional parameters and multiple return types
  - Dict and List type specifications with element types

- **Model Documentation**: Enhanced Pydantic model documentation
  - Class-level docstrings explaining purpose and usage
  - Field-level documentation for important attributes
  - Config class documentation

Files Updated:
- Core application: `app/main.py`, `app/config.py`, `app/database.py`, `app/auth.py`
- Routes: `app/routes/auth.py`, `app/routes/admin.py`, `app/routes/public.py`
- Utilities: `app/utils/images.py`, `app/utils/pagination.py`, `app/utils/__init__.py`
- Templates: `app/templates_helpers.py`
- Data: `app/seed_data.py`
- Models: `app/models/rootbeer.py`, `app/models/review.py`, `app/models/flavor_note.py`,
  `app/models/admin_user.py`, `app/models/metadata.py`
- Scripts: `generate_icons.py`

Benefits:
- Improved IDE autocomplete and type checking
- Better code maintainability and readability
- Easier onboarding for new developers
- Foundation for automated documentation generation (Sphinx)
- Enhanced type safety and error detection

This work prepares the codebase for comprehensive test suite development by
ensuring all code is well-documented and type-annotated.
