fix: Generate PWA icons and fix deprecated meta tags

This commit fixes PWA installation issues by generating all required icon
sizes and updating deprecated meta tags.

Bug Fixes:
- Generated all required PWA icon sizes (72x72 through 512x512)
  - Resolved 404 errors for missing icons that prevented Chrome install prompt
  - Icons now properly display in Chrome's address bar install option
- Fixed deprecated meta tag warning
  - Added mobile-web-app-capable meta tag (replaces deprecated apple-mobile-web-app-capable)
  - Updated both public and admin base templates

Added:
- Icon generation script (generate_icons.py)
  - Creates all required PWA icon sizes with root beer theme
  - Uses Pillow for image generation
  - Generates icons with root beer emoji on sassafras background
  - Can be run with: uv run python generate_icons.py

Files Changed:
- app/templates/base.html: Added mobile-web-app-capable meta tag
- app/templates/admin/base.html: Added mobile-web-app-capable meta tag
- app/static/icons/: Generated all 8 required icon sizes
- generate_icons.py: New icon generation utility script
- CHANGELOG.md: Added version 0.2.3
- ROADMAP.md: Marked PWA icons as completed
- README.md: Updated PWA feature description
