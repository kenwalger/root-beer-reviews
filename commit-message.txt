Add comprehensive test suite with 76% code coverage

This PR adds a full pytest-based test suite with 108 tests covering all major functionality, achieving 76% code coverage. The test suite includes unit tests, integration tests, and proper async handling.

## Test Coverage

### Test Suite Overview
- **Total Tests**: 108 tests (all new for this PR)
- **Unit Tests**: Models, utilities, authentication
- **Integration Tests**: All admin routes, public routes, auth flows
- **Coverage**: 76% code coverage (up from 0%)

### Test Categories Added
- **Admin CRUD Operations**: Create, read, update, delete tests for root beers and reviews
- **Image Management**: Upload, delete, and set primary image tests (with S3 mocking)
- **Flavor Note Management**: List, create, and delete tests
- **Metadata Management**: Colors and serving contexts CRUD tests
- **Admin Account Management**: Password change with validation tests
- **Auth Error Paths**: Invalid token handling and exception testing
- **Public Routes**: Homepage, filtering, pagination, public views
- **Model Validation**: Pydantic model tests with edge cases

### Test Infrastructure
- Set up pytest with httpx.AsyncClient for proper async testing
- Test fixtures for database, admin users, and sample data
- MongoDB Atlas integration with separate test database
- S3 mocking for image upload tests
- Coverage reporting with HTML output

## Code Quality Improvements

### Documentation
- All Python files now use Sphinx-style docstrings
- Comprehensive type hints throughout the codebase
- Function and class documentation with parameter descriptions

### Deprecation Fixes
- Replaced all `datetime.utcnow()` with `datetime.now(UTC)`
- Migrated Pydantic `class Config` to `model_config = ConfigDict(...)`
- Replaced deprecated `json_encoders` with `field_serializer`
- Fixed Starlette `TemplateResponse` parameter order (request first)

### Route Fixes
- Fixed `create_color` and `create_serving_context` routes to properly use `Form(...)` for form data
- Removed unused `request` parameter from metadata routes
